1. Which resource is the single biggest waste, and why?
prod-db-main at $1,200/mo. Idle 80% of the time means you are paying for 5x more capacity than you actually use. cdn-old is also 100% waste but costs less.
2. What would you do to fix dev-vm-01?
Add an auto-shutdown schedule: off at 7pm weekdays, off all weekend. Saves ~$320/mo with zero impact on dev work. Tag it environment:dev so the policy applies to future dev VMs automatically.
3. To save $2,025 (30%), which resources would you optimise?
Delete cdn-old ($600) + right-size prod-db-main ($600) + delete storage-backup ($450) = $1,650. Add dev-vm-01 evening shutdown (~$375). Total: ~$2,025 — without touching the two justified resources.
4. What tags would prevent this next month?
Enforce: owner, environment, purpose, expiry, auto-shutdown. Block resource creation if tags are missing. Alert within 24 hours on non-compliant resources.
5. What would you ask the data science team about ml-compute?
"Is this workload predictable enough for Reserved Instances? A 1-year commitment saves 30–40% (~$750–$1,000/mo) with no workflow change. Also — does it run 24/7 or only during working hours?"
